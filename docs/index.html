SOLACE App Engine

SOLACE App Engine (SAE) is a system for storing and running multiple stand-alone Javascript applications all under one roof. Each of the apps that SOLACE App Engine can "run" is stored inside of the app directory in a separate folder. All of the apps are stand-alone. They can run by themselves, and function indepent from SOLACE App Engine. If you go directly to /sae/app/name/, you should be able to run the application by itself. Everything the Javascript app needs should be self-contained within the directory (scripts, stylesheets, templates, HTML files, images, ...everything!) Each of the applications is separate and unique.

So, why SAE? The purpose of SOLACE App Engine was, originally, to provide a common set of libraries and functions for the six core apps being used by SOLACE Network. Each of these apps can function indepently, but, instead of having 6 separate copies of every Javascript library or image, it made more sense to have a single repository for all of them. Thus, SOLACE App Engine was born as a way to provide a common interface to access each of the six separate Javascript applications. Early iterations of SAE had a built-in menu system. But, it was more efficient to just convert the SOLACE Menu to be a stand-alone app and include with the six core apps. But, I kept thinking of ideas of new apps (or add-ons). I could keep exanding the "core apps" and, eventually, end up with like fifty of them but, that didn't make sense. The SOLACE codebase was already getting large and difficult to manage with only a few apps.

This was the impetus for turning SOLACE App Engine into something stand alone too. The App Engine loads, checks the environement, and displays the SOLACE Menu app dynamically on any-sized device. From here, you can load the six core SOLACE apps: Directory, Education, Garden, Home, Library, Network. There is a button at the bottom of the screen which shows installed 3rd party apps. By 'installed', it just means there is a copy of the directory on the user's computer. There are stand-alone Javascript apps that run from a single directory in a browser using a web server (like Apache or ). There is also a button at the bottom of the screen which show additional apps available for installation. You can navigate the categories, click an app, view screenshots, description, reviews, ratings, and install or view status.

SOLACE App Engine (SAE) does contain a set of "shared" or common libraries including: jQuery, Bootstrap, Boxicons, Bootrap Icons, etc. Each of the stand alone apps available on SOLACE is able to use these shared resources but, they are not required to. If a developer wants to include their own library instead of using the shared resources, they're free to do so. (Just remember: app size is displayed to the end-user before installation). The core apps from SOLACE and the SOLACE Menu all use the shared resources from SAE rather than duplicating the files in each sub-directory.

Developers can write their apps to use either the SAE libraries (under "/lib/vendor/") or a stand-alone copy of the library inside the folder where the app is running. You can add as many sub-folders and libraries as you need to your app as long as the total size of the folder stays below 100 MB (zipped). You can even write your code to toggle between SAE libaries and stand-alone dev libraries which are self-contained in the app, or you can make it a user preference. It's up to you, the developer.

User Interface (UI)

Reserved Words:

<div id="ui"></div>

The main UI for SAE has a div with an id called 'ui'. This is the 'target div' for the entire SAE application. This id name ('ui') is a reserved SOLACE word which should be exclusive to the SOLACE App Engine. If multiple Javascript apps are all using the same IDs, it could result in Javascript errors. To avoid this problem, try using alternative div id names in your apps (e.g. 'wrapper', 'container', 'box', 'view', etc.). It would prevent any future confusion if you just named your own div something different. In the event of a conflict, the code will give preference to SAE functions first. This could result in Javascript errors in your 3rd party app running inside an iframe in SAE.

Extra Resources (vendor)

The /lib/vendor/ directory contains copies of 3rd party scripts which come with their own license. They are provided here for your convenience. But, always feel free to grab the latest versions directly from the source and use your own code libraries. We try to keep up with version changes but, sometimes we fall behind. If you notice a 3rd party library which needs to be updated, please let us know.
